Data Storage

Privacy and personal data protection is a core design principle of the application. For this reason, user data is stored locally on the device only, and we intentionally avoid collecting or retaining personally identifiable information.

In particular, the app does not store sensitive details such as full name, passport information, seat number, or any other booking-specific identifiers. Instead, it stores only generalized, non-identifying preference/personality attributes that are required to personalize the experience, and these remain exclusively on the user's device.

User Control

Users can delete all locally stored data at any time through the in-app settings. Additionally, uninstalling the application removes all stored information, since no user data is persisted on our servers.

External Services and Secure Transport

When certain features require external services, for example travel document verification or routing, only the minimum necessary inputs are sent to the backend and then forwarded to the respective third-party APIs.

All such communications take place over HTTPS POST requests, which provides transport-layer encryption end-to-end. We do not intercept, modify, or inspect the encrypted channel beyond relaying the required requests and normalizing the responses for the client.
